# alexajuno

Opinionated developer workflow skills for Claude Code.

## Install

```
claude plugin add github:alexajuno/claude-code-plugin
```

## Skills

| Skill | What it does |
|-------|-------------|
| **git-workflow** | Commit message format (`prefix: what`), branch naming (`feat/`, `fix/`), PR body with Motivation/Summary/Test Plan sections, no AI attribution |
| **update-pr** | Update PR title/body, add comments, reply to review feedback via `gh` API |
| **github-release** | Create GitHub releases with semver bump suggestions and auto-generated notes |
| **schema-audit** | Audit Laravel migrations for UUID usage, soft deletes, cascade safety, index strategy, FK behavior |
| **doctl** | DigitalOcean App Platform CLI reference — deployments, logs, app management |
| **ui-engineering** | Component-driven development with design tokens for frontend work (any framework) |
| **playwright-hygiene** | File cleanup, dev server detection, and conventions for Playwright MCP tools |
| **web-pdf-reader** | Read PDFs from web URLs — arxiv papers, docs, any hosted PDF |
| **vibe-learning** | Turn coding sessions into learning experiences with light/medium/deep explanation modes |
| **lyrics** | Look up song lyrics and explain their meaning |

## Philosophy

These skills encode real workflow preferences, not generic best practices. They're opinionated by design — if your team has different conventions, fork and adapt.
